## Quick Start

### 1. Validate DPF Tool Accuracy

```bash
cd "DPF Validation and Testing"
python calculate_dpf_accuracy.py    # Calculate DPF accuracy against ground truth
```

### 2. Generate Code with LLMs

```bash
python generate_code.py             # Generate code using different LLM models
```

### 3. Analyze LLM-Generated Code Patterns

```bash
cd LLM_Code_Analysis
python analyze_llm_patterns.py      # Run analysis
python export_llm_analysis.py       # Generate CSVs and insights
python generate_report.py           # Create markdown report
```

See `LLM_Code_Analysis/README.md` for detailed documentation.

## Key Research Findings

### LLM Pattern Analysis
- **17.5%** of LLM-generated code contains design patterns
- **Builder** is most common (7.17% of all files)
- **gpt_oss_120b** performs best (39% detection rate)
- Significant model variance: 10% to 39%

### DPF Tool Validation
- **100%** accuracy on ground truth repositories
- Successfully validates against 463 Java files
- Correctly identifies all 22 pattern types tested

## Tools Used

- **DesignPatternFinder (DPF)** - Static analysis tool for detecting design patterns in Java code
- **LLMs Tested**: gemma_2_2b_it, gpt_oss_120b, llama_3_1_8B_Instruct, llama_3_2_1B_Instruct, qwen2_5_7b_instruct, qwen2_5_coder_32b

## Research Objectives

1. Evaluate which design patterns are most commonly generated by LLMs
2. Compare pattern generation capabilities across different LLM models
3. Validate the accuracy of pattern detection tools
4. Understand LLM code generation characteristics and biases
